Use UTM parameters to add tracking values to ads launched through Adtool. Defaults can be set at three layers: organization-wide default, selected ad account default, and batch-level override.

UTMs help analytics tools identify where traffic came from, which campaign generated it, and which ad or ad set was clicked. In Adtool, saved UTM defaults are applied when ads are prepared for launch, unless you adjust the UTM parameters for a specific batch.

## How the default layers work

| **Layer** | **Where it is set** | **What it affects** |
| --- | --- | --- |
| Organization-wide default | `Settings` → `Default settings` → `UTM parameters` | Applies across all current and future ad accounts in the organization after saving. |
| Selected ad account default | `Settings` → select an ad account → `UTM parameters` | Applies only to the selected ad account after saving. It overrides the organization-wide default for that selected account. |
| Batch-level override | `Launch` → `Additional settings` while creating a batch | Changes UTM parameters only for the current batch. It does not change saved organization-wide or selected ad account defaults. |

<Note>
  UTM parameters only help tracking when the launched ads send traffic to a URL where those parameters can be used by your analytics setup.
</Note>

## Set the organization-wide default

Use the organization-wide default when every current and future ad account in the organization should start with the same UTM setup.

<Steps>
  <Step title="Open Settings" titleSize="h4">
    Go to `Settings` in the left navigation bar.

      alt="Settings page in Adtool"
      title="FB IG PAGE1 Pn"

    />
  </Step>
  <Step title="Open Default settings" titleSize="h4">
    In the left navigation bar, find the same section where ad accounts are selected. Click the top option, `Default settings`.
  </Step>
  <Step title="Find UTM parameters" iconType="regular" titleSize="h4">
    Find the `UTM parameters` section.

    <img
      src="/images/select_utm-paramter.png"

  </Step>
  <Step title="Set the organization-wide default" titleSize="h4">
    Enter the UTM string you want Adtool to use, then save the page/settings. When saved here, the default applies across all current and future ad accounts in the organization.

## Set the selected ad account default

Use the selected ad account default when one selected account should use a different UTM setup than the organization-wide default.

<Steps>
  <Step title="Open Settings" titleSize="h4">
    Go to `Settings` in the left navigation bar.
  </Step>
  <Step title="Select the ad account" titleSize="h4">
    In the left navigation bar, find the section where ad accounts are selected. Choose the ad account you want to adjust instead of `Default settings`.

    <img
      src="/images/Select_ad_accounts.png"
      alt="Ad account selector in Settings"
      title="Select Ad Accounts Pn"
      style={{ width:"100%" }}
    />
  </Step>
  <Step title="Find UTM parameters" iconType="regular" titleSize="h4">
    Find the `UTM parameters` section.
  </Step>
  <Step title="Set the selected ad account default" titleSize="h4">
    Enter the UTM string for that selected account, then save the page/settings. When saved here, the selected ad account default applies only to that account and overrides the organization-wide default for that selected account.
  </Step>
</Steps>

## Change UTM parameters for one batch

Use the batch-level override when only the batch you are creating should use a different UTM setup.

<Steps>
  <Step title="Go to Launch" titleSize="h4">
    Go to `Launch`.
  </Step>
  <Step title="Open Additional settings" titleSize="h4">
    While creating a batch, open `Additional settings`.
  </Step>
  <Step title="Change UTM parameters for the batch" titleSize="h4">
    Adjust the UTM parameters for that batch.
  </Step>
</Steps>

Changing UTM parameters at batch level does not change the saved organization-wide or selected ad account defaults. It only changes the batch being created at that moment.

## Recommended setup

Use a consistent structure that identifies the platform, traffic type, campaign, ad set, and ad. A typical setup is:

```text
utm_source=facebook&utm_medium=paid_social&utm_campaign={{campaign.name}}&utm_term={{adset.name}}&utm_content={{ad.name}}
```

| **Parameter** | **Typical purpose** |
| --- | --- |
| `utm_source` | Identifies the traffic source, such as `facebook`. |
| `utm_medium` | Identifies the channel or traffic type, such as `paid_social`. |
| `utm_campaign` | Identifies the campaign. Dynamic values can use `{{campaign.name}}`. |
| `utm_term` | Often used for the ad set. Dynamic values can use `{{adset.name}}`. |
| `utm_content` | Often used for the ad. Dynamic values can use `{{ad.name}}`. |

## Before you save

Check these items before saving defaults or launching a batch:

| **Check** | **Why it matters** |
| --- | --- |
| You are editing the right layer | Organization-wide defaults affect all current and future ad accounts, selected ad account defaults affect one selected account, and batch-level overrides affect only the current batch. |
| Parameter names are spelled correctly | Analytics tools usually expect names like `utm_source`, `utm_medium`, `utm_campaign`, `utm_term`, and `utm_content`. |
| Values use the right separators | Use `&` between parameters and `=` between each parameter name and value. |
| Dynamic values match your reporting needs | `{{campaign.name}}`, `{{adset.name}}`, and `{{ad.name}}` make reporting easier when names are clear. |

## Common cases

<Expandable title="I want one UTM setup for every current and future account">
  Set the organization-wide default from `Settings` → `Default settings` → `UTM parameters`.
</Expandable>

<Expandable title="I want a different UTM setup for one selected account">
  Go to `Settings`, select the ad account you want to adjust, and update `UTM parameters` for that selected account.
</Expandable>

<Expandable title="I only want to change one launch batch">
  Do not change the saved defaults in `Settings`. Go to `Launch`, open `Additional settings`, and adjust UTM parameters for that batch.
</Expandable>
